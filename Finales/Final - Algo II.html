GRAFOS

Maneras de representación:

  • Matriz de adyacencia, ocupa mucha memoria pero es más rápida.
  • Lista de adyacencia, se puede usar memoria dinamica pero el recorrido es mas lento en comparacion con la matriz.
  • Matriz de incidencia, no se usa. Es mas compleja.

Se puede pedir que a partir de alguna se haga otra, es decir, que a partir de la matriz de adyacencia se haga la lista de incidencia por ejemplo.

Ciclo: Recorrido de al menos dos aristas que empieza y termina en el mismo lugar. Grafo no dirigido no hay ciclos
Camino: Recorrido a traves del grafo.
Camino simple: Camino que pasa una vez por un camino

Recorridos

Significa:

  • Proceso de busqueda a traves de un grafo.
  • Visitar el grafo

Para hacer estos algoritmos, se agarra la copia del grafo y se trabaja sobre el mismo.

BFS (Breadth First Search)

Recorrido en
Dado un nodo, se recorren sus os.

Se lo hace utilizando una la:

  • lamos un vertice
  • Quitamos un vertice, visitamos.
  • lamos los adyacentes no visitados.
  • Repetir.

DFS (Depth First Search)

Recorrido en profundidadanchura
Dado un nodo, se recorren sus vecinhijos.

Se lo hace utilizando una picola:

  • ApiEncolamos un vertice
  • Quitamos un vertice, visitamos
  • ApiEncolamos los adyacentes
  • Repetir

Orden topologico

Especie de orden para recorrer los nodos.
Hay un algoritmo que dado un grafo te devuelve un vector con los elementos en el orden que se deberá recorrer. Ese orden se llama Orden topologico

No hay un único orden topologico.
Solo para grafos que no tienen ciclos (Grafo aciclico) y dirigidos.

Sea el caso en que los nodos son tareas, el orden topologico se utiliza para determinar en que orden se ejecutan las tareas.

Algoritmo:

  • Se arranca por el nodo que no tiene ningún entrante.
  • Se anota el nodo en una lista de nodos.
  • Se elimina el nodo del grafo, junto con sus aristas.

Maximal

  • El nodo raiz es el mayor -> tiene el elemento con valor maximo. Por lo tanto sus hijos van a ser menores.
  • No hay relacion entre hermanos acerca del orden
  • Verticalmente si existe la relacion. Los de mas arriba son mayores que los de mas abajo.
  • Arbol, casi completo --> Pq se llena de izq a derecha, por niveles. Si estan todos los niveles llenos es completo. El unico nivel que puede aparecer con nodos de menos es el ultimo nivel.
    -Adicionalmente los nodos que pueden no estar son los de la derecha.
    PARA QUE SEA CASI COMPLETO --> SOLO DEBEN FALTAR NODOS DE LA DERECHA.

#define ERROR -1 
#define EXITO 0
#define MAX_ELEMENTOS 100

typedef struct{
   int* vector:
   int tope;
}heap_t;

heap_t* heap_crear(){
   return calloc(1, sizeof(heap_t));
}

void swap(int* vector, int i, int j){
   int aux = vector[i];
   vector[i] = vector[j];
   vector[j] = aux; 
}
int posicion_padre(int n){
   return (n-1)/2;
}
int posicion_hijo_derecho(int n){
   return 2*n+2;
}
int posicion_hijo_izquierdo(int n){
   return 2*n+1;
}

void sift_up(heap_t* heap, int n){
   if(n==0){
   	return;
   }
   int padre = posicion_padre(n);
   if(padre < heap->vector[n]){
   	swap(heap->vector, n, padre);
   	sift_up(heap, padre);
   }
}

int heap_extraer_raiz(heap_t* heap){
   if(!heap){
   	return ERROR; 
   }
   if(heap->tope == 0){
   	return 0;
   }
   int valor = heap->vector[0];	
   
   swap(heap->vector, 0, heap->tope-1);
   heap->tope--;
   
   if(heap->tope != 0){
   	sift_down(heap, 0);
   }
   
   return valor;
}

int heap_insertar(heap_t* heap, int elemento){
   void* aux = realloc(heap->vector, sizeof(int)*heap->tope+1);
   if(!aux){
   	return ERROR;
   }
   heap->tope++;
   heap->vector = aux;

   heap->vector[heap->tope-1] = elemento;	
   sift_up(heap, heap->tope-1);

   return EXITO;
}

void sift_down(heap_t* heap, int n){
   int pos_der = posicion_hijo_derecho(n);
   int pos_izq = posicion_hijo_izquierdo(n);
   int pos_mayor = pos_izq;
   
   if(pos_izq >= heap->tope){
   	return;
   }

   if(pos_der < heap->tope){
   	if(pos_der < pos_izq){
   		pos_mayor = pos_der;
   	}
   }
   if(heap->vector[n] > heap->vector[pos_mayor]){
   	swap(heap->vector, n, pos_mayor);
   	sift_down(heap, pos_mayor);
   }
}

void heap_destruir(heap_t* heap){
   if(heap){
   	free(heap->vector);
   }
   free(heap);
}

void heapify(int* vector, int n){
   heap_t*  heap = heap_crear();
   heap->vector = vector;
   heap->tope = n;
   
   int primero = posicion_padre(n-1);
   while(primero>=0){
   	sift_down(heap, primero);
   	primero--;
   }
   
   return heap;
}


void heap_sort(int* vector, int n){
   heap_t* heap = heapify(vector, n);
   while(n>0){
   	heap_extrar_raiz(heap);
   	n--;
   }
   
   free(heap);
}

int main(){
   heap_t* heap = heap_crear();

   heap_insertar(heap, 1);
   heap_insertar(heap, 10);
   heap_insertar(heap, 25);
   heap_insertar(heap, 40);
   heap_insertar(heap, 23);
   heap_insertar(heap, 5);
   heap_insertar(heap, 7);
   heap_insertar(heap, 12);
   heap_insertar(heap, 17);
   heap_insertar(heap, 23);

   heap_destruir(heap);

   //heap sort
   int vector[] = [7, 8, 32, 2, 9, 11, 1, 25, 18, 42];

   int cantidad = sizeof(vector)/sizeof(int);

}

HASH

Tabla de hash:

  • Estructura que contiene valores
  • Puedo hallar un valor a partir de una clave

Función de hash:

  • Es una función que transforma claves en un numero asociado.

Colisiones:

  • Pueden haber mas claves que espacios en la tabla de hash
    • Claves distintas dan el mismo valor de hash.
  • Se resuleven distinto segun el tipo de hash.

Tipos de hashes

  • Hash cerrado --> Direccionamiento abierto1

    • Esta almacenado dentro del hash.
    • Tamaño de la tabla >= nro de claves.
    • Colisiones:
      • Se resuelve recorriendo el array hasta encontrar el proximo espacio libre.
      • Tipos de metodos de busqueda:
        • Probing lineal -> Busca el siguiente espacio libre inmediato
        • Probing cuadratico -> Intenta insertar
        • Hash doble -> aplicar una segunda función de hash a la clave cuando hay colisión
      • Cuando nos quedamos sin espacio en la tabla, agrandamos la misma y tenemos que rehashear todos los elementos ya que no podemos asegurar que esta en el mismo lugar.

  • Hash abierto --> Direccionamiento cerrado2

    • Esta almacenado por fuera del hash.
    • O(n), n la cantidad de elementos que colisionaron.
    • Colisiones:
      • Se resuleve mediante chaining o encadenamiento
      • Cada vez que haya una colision, es decir que la funcion de hash me de el mismo indice para diferentes valores, ese indice va a representar un puntero a una lista.

Funcion de hash para que entre en la tabla hay que aplicarle la operacion modulo -> operacion resto. El resto de la division entera. Es decir, que el tamaño de la tabla es 100 entonces el numero que te da la funcion de hash, dividimos por 100.

a) El algoritmo de Floyd - Marshall se encarga de encontrar la distancia minima entre todos los pares de vertices del grafo.
Toma todos los pares de vertices para llegar desde un vertice hasta otro, comparando las distancias conocidas con otras distancias que existen para llegar al mismo vertice a traves de otro camino,.
Es decir, para llegar de A->B se puede hacer A->B o A->C->B, por ejemplo.
La idea es ir armando una tabla con estas distancias hasta que finalmente se terminen de recorrer todos los caminos posibles y por lo tanto quede la distancia minima entre estos vertices.
